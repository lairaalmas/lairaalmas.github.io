Namespace: state

game.state

Handles game states When an state is associated with an object, its preload(), create() and update() functions will be executed acconrding to these rules: * preload() : first function to be called when state is called. Loads media. Runs only once.(optional) * create() : called right after preload(). Where the main code goes. Runs only once. * update() : called right after create(). Is iteratively called by game loop until end of the state. (optional)
